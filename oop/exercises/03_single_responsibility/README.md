# 03: 責務の分離

## 題材：日報システム

日報を管理し、複数の形式で出力できるシステムを作ってください。

### 要件

- 日報は **日付** と **作業内容（配列）** を持つ
- 日報を **プレーンテキスト形式** で出力できる
- 日報を **マークダウン形式** で出力できる

### 出力例

#### プレーンテキスト形式

```
=== Report: 2025-01-15 ===
- Implement user authentication
- Code review
- Team meeting
```

#### マークダウン形式

```
# Report: 2025-01-15

- Implement user authentication
- Code review
- Team meeting
```

### 考えてほしいこと

この問題を解く前に、以下の問いについて考えてみてください：

1. **「日報のデータ管理」と「出力形式」は同じ責務？別の責務？**
   - 日報の日付や作業内容を保持することと、それを整形して表示すること、同じクラスでやるべき？

2. **新しい出力形式（HTML、PDF等）を追加したくなったら？**
   - 1つのクラスに全部書いた場合、どこを変更する？
   - 分けて書いた場合、どこを変更する？

3. **「変更の理由」で考える**
   - 日報の項目（日付、作業内容）が変わるのはどんなとき？
   - 出力フォーマットが変わるのはどんなとき？
   - それは同じタイミング？違うタイミング？

### ファイル

以下のファイルを作成してください：
- `daily_report.rb` - 日報クラス
- `text_formatter.rb` - プレーンテキスト形式で出力するクラス
- `markdown_formatter.rb` - マークダウン形式で出力するクラス

（クラス設計は自由です。上記は一例）

### テストの実行

```bash
cd oop
rspec spec/03_single_responsibility/
```

### キーワード

調べてみよう：
- 単一責任の原則（Single Responsibility Principle / SRP）
- 「変更の理由」による責務の分離
- 凝集度（Cohesion）
